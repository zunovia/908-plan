# Zero — 使い方マニュアル

> 声の鏡。30秒の声から、無意識のパターンを映す。

---

## Part 1: ユーザーガイド

### 1. Zeroとは

Zeroは「声の鏡」です。毎日30秒だけ声を録音し、言葉の内容ではなく**声そのもの**——エネルギー、明瞭さ、抑揚、テンポ——を分析します。日々の微細な変化を可視化し、自分でも気づかなかった無意識のパターンを映し出すアプリです。

AIが言葉を解釈するのではありません。声の物理的な特徴だけを数値化し、あなた自身が気づきを得るための「鏡」として機能します。

---

### 2. 初回セットアップ（オンボーディング）

アプリを初めて起動すると、6ステップのオンボーディングが始まります。

| ステップ | 画面 | 内容 |
|---|---|---|
| — | スプラッシュ | 15秒のイントロアニメーション。匿名アカウントが自動作成されます |
| 1/6 | 哲学 | Zeroの思想を紹介。「始める」で次へ進むか、「この言葉について」で湖のメタファーを読めます |
| 2/6 | メタファー（任意） | 湖のメタファーによる補足説明。「始める」で次へ |
| 3/6 | 仕組み | 声の分析方法を説明（言葉ではなくピッチ・テンポ・変動を見る） |
| 4/6 | マイク許可 | マイクへのアクセスを許可してください。拒否した場合は「設定を開く」から許可できます |
| 5/6 | 通知 | レポート通知の許可。「あとで」でスキップ可能 |
| 6/6 | 初回録音 | 「今、何を感じていますか？」——録音ボタンを押すと最初の録音が始まります |

### 3. ホーム画面

ホーム画面には以下が表示されます:

- **連続記録日数** — 「◯日連続」または「今日から始めよう」
- **ミニトレンドチャート** — 7日以上の録音データがある場合に表示
- **日替わり問いかけ** — 20種類の問いかけが日ごとにローテーション（例:「今、頭の中にあることを話してください」「今日の朝、最初に感じたことは何でしたか」）
- **録音時間の選択** — 3つから選べます:
  - ひと息15秒
  - いつもの30秒（デフォルト）
  - じっくり60秒
- **録音ボタン** — タップすると録音画面に移動

---

### 4. 録音フロー

録音は5つのフェーズで構成されます。

#### フェーズ1: 録音

選択した秒数（15/30/60秒）の間、自由に話してください。波形ビジュアライザーが声の大きさに反応します。タイマーが終了すると自動的に次のフェーズへ進みます。

#### フェーズ2: 呼吸

3秒間の短い呼吸タイム。脈打つドットのアニメーションが表示されます。自動的に次へ進みます。

#### フェーズ3: ミニインサイト

今回の録音から得られた1つの気づきが表示されます。4つの指標（テンポ・エネルギー・明瞭度・抑揚）が日替わりでローテーションします。「次へ」をタップして進みます。

#### フェーズ4: 自己評価

今の気分を選んでください（スキップも可能）:

| 選択肢 | 意味 |
|---|---|
| 穏やか | 落ち着いている |
| 普通 | いつも通り |
| 揺れている | 不安定さを感じる |
| わからない | 判断できない |

#### フェーズ5: 静かな言葉

12秒間、一つの問いかけがフェードイン→フェードアウトします:

> 「今日の声は、昨日と似ていましたか？」

その後、自動的にホーム画面に戻ります。

### 5. レポート

#### 週間レポート

毎週のデータをまとめて表示します:

- **トレンドチャート** — エネルギー・明瞭度・抑揚・テンポの4指標を折れ線グラフで表示
- **ハイライト** — 最も生き生きしていた日、最も静かだった日
- **問いかけ** — 10のルールから最も顕著なパターンを検出し、それに基づく問いかけを生成

#### 月間レポート

月全体のデータに加え、先月との比較を表示します:

- **トレンドチャート** — 月間の推移
- **ハイライト** — 月間のベスト/ワーストデー
- **先月との比較** — 最も変化の大きかった指標とその解釈

---

### 6. 履歴

カレンダー形式で録音した日を確認できます。

- **●** = 録音あり、**○** = なし
- 日付をタップすると、その日の録音詳細（時刻、指標値、自己評価）が表示されます
- 録音がまだない場合は「まだ録音がありません。最初の声を記録してみましょう。」と表示されます

### 7. 設定

#### アカウント

- **ログイン状態** — 匿名またはアカウント連携済み
- **アカウント連携** — Apple（iOSのみ）またはGoogleアカウントと連携可能。連携するとデバイスを変更してもデータを引き継げます

#### 音の設定

- **効果音** — ボタンタップ音・遷移音のON/OFF
- **バイノーラル** — ON/OFF

#### 通知

- **レポート通知** — 週間/月間レポート完成時の通知ON/OFF
- **リマインダー** — 毎日の録音リマインダー時刻を設定（キャンセルで解除）

#### テーマ

- **ダーク** / **ライト** / **自動**（システム設定に追従）

#### その他

- **デモを見る** — サンプルレポートを表示
- **プライバシーポリシー** — ボトムシートで表示
- **利用規約** — ボトムシートで表示
- **休む（お休みモード）** — 一時的に通知を停止。再度タップで解除
- **アカウント削除** — すべてのデータが完全に削除されます（取り消し不可）

**設計パターン:**

- **`Future.microtask()` パターン** — `build()` 内で非同期データをロードする際、直接 `state = ...` を呼ぶと例外になるため `Future.microtask(() => _loadFromStorage())` で遅延実行する（`SettingsNotifier`, `HistoryNotifier`, `ReportNotifier` 等で使用）
- **Sentinel パターン** — nullable フィールドの `copyWith` で「明示的に null をセット」と「変更なし」を区別するため `static const _sentinel = Object()` を使用（`HistoryState`, `SettingsState`）

#### ルーティング: GoRouter

`lib/app.dart` で定義。認証状態とオンボーディング完了状態を `watch` し、リダイレクトガードを適用:

```
redirect ロジック:
- `/` (スプラッシュ) → 常に許可
- `/demo*` → 常に許可（認証不要）
- 未認証 → `/` にリダイレクト
- 認証済み + オンボーディング未完了 → `/onboarding/philosophy` にリダイレクト
```

**ShellRoute（BottomNav付き）:** `/home`, `/reports`, `/history`, `/settings`

#### データ層

| ストレージ | 用途 | 実装 |
|---|---|---|
| Hive (`hive_ce`) | 録音データ（RecordingModel） | Box名 `'recordings'`、キー = millisecondsSinceEpoch |

### 5. 音声解析パイプライン

```
RecorderService (WAV 44100Hz mono)
  → ファイルパス取得
  → VoiceAnalyzer.analyze(filePath)
    → WAVパース (wav パッケージ)
    → STFT (window=2048, hop=512, Hanning窓)
    → FFT (fftea パッケージ: realFft → Float64x2List)
    → 4指標を算出
  → VoiceMetrics
```

#### VoiceMetrics 4指標

| 指標 | 算出方法 | 範囲 |
|---|---|---|
| **energy** | RMS → dBFS変換 → `(dBFS + 60) / 60` | 0.0–1.0 |
| **clarity** | スペクトル重心（各フレーム） → 平均 → `(centroid - 500) / 2500` | 0.0–1.0 |
| **expressionRange** | F0検出（80–400Hz帯ピーク） → 標準偏差 → `stdDev / 80.0` | 0.0–1.0 |
| **tempo** | エンベロープ（50msウィンドウ） → ピーク検出 → `peakCount / durationSec` | 1.0–8.0 |

フォールバック値（解析失敗時）: `energy=0.5, clarity=0.5, expressionRange=0.4, tempo=3.0`

### 6. InsightEngine

`lib/core/insights/insight_engine.dart` — 純粋なDart静的クラス。LLM不使用。

#### ミニインサイト（録音直後）

`dayOfYear % 4` でローテーション:

| スロット | 表示指標 | 例 |
|---|---|---|
| 0 | テンポ | 「今日の声のテンポは、1秒あたり3.2音節でした。自然なペースですね。」 |
| 1 | エネルギー | 「今日の声のエネルギーは72%でした。力強さのある声ですね。」 |
| 2 | 明瞭度 | 「今日の声の明瞭度は65%でした。いつもの自分らしい声ですね。」 |
| 3 | 抑揚 | 「今日の声の抑揚スコアは45%でした。静かで落ち着いた声ですね。」 |

#### 週間問いかけ（10ルール + 顕著度スコア）

全ルールを評価し、最高スコアの問いかけを選択:

| # | ルール | トリガー条件 |
|---|---|---|
| 1 | energy_spike | 連続日間のエネルギー差 > 0.15 |
| 2 | expression_rich | 最大抑揚 > 0.5 |
| 3 | clarity_trend | 後半 vs 前半の明瞭度差 > 0.1 |
| 4 | tempo_variation | テンポ標準偏差 > 0.5 |
| 5 | tempo_consistency | テンポ標準偏差 < 0.2 |
| 6 | energy_clarity_diverge | エネルギーと明瞭度が逆方向に推移 |
| 7 | low_energy_week | 全日エネルギー < 0.35 |
| 8 | high_expression_all | 全日抑揚 > 0.4 |
| 9 | weekend_weekday_diff | 平日と週末のエネルギー差 > 0.15 |
| 10 | stable_week | 全指標の標準偏差が小さい |

フォールバック: 「今週の声に、耳を澄ませてみてください。」

#### 月間比較

- 今月と先月の4指標平均を比較（テンポは `/7.0` で正規化）
- 最大差分の指標について方向性と変化の大きさをコメント
